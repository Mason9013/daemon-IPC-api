How it works
01
Named FIFO
Client writes a unique ID to the well-known REQUESTS pipe. The daemon reads it in a tight loop.
02
Double Fork
Daemon double-forks to spawn an orphaned worker, fully detached from the daemon's process group.
03
Private FIFOs
Worker opens clientid.0 and clientid.1 for bidirectional communication with the client.
04
execv()
Worker replaces itself with the demo executable via execv(), inheriting the FIFO file descriptors as stdin/stdout.
